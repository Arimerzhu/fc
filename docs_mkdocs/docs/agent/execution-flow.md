# 五阶段执行流

fc 强制 AI Agent 按五阶段流程执行所有设计任务。**跳过任何一步都算失败。**

## 阶段概览

| 阶段 | 名称 | 做什么 |
|------|------|--------|
| 1 | 工具选型 | 列出所有命令组和命令 |
| 2 | 任务拆解 | 拆为原子步骤，每步一条 CLI 命令 |
| 3 | 坐标与依赖 | 计算坐标、标注依赖元素、拓扑关系 |
| 4 | 依赖校验 | 检查所有依赖合法，无悬空引用和循环依赖 |
| 5 | 命令输出 | 按顺序输出 CLI 命令 + 注释 |

## 核心规则

1. 只使用 FreeCAD CLI 命令，不编造命令
2. 所有元素唯一命名：`{类型}_{序号}`（如 `Box_001`）
3. 只引用已创建的元素
4. 遵循 FreeCAD 绘图顺序：基准 → 基础几何 → 布尔 → 约束 → 导出
5. 不合并多个操作到一条命令
6. 每批最多 10 条命令，执行完反馈结果
7. 始终使用 `--json` 输出
8. 多步工作流使用 `--project` 保持会话

## 禁止规则

- ❌ 不在 Z=0 以下创建几何体
- ❌ 对不存在的元素做布尔运算
- ❌ 圆没有附着平面
- ❌ 跳过任何阶段
- ❌ 一次超过 10 条命令

## 输出模板

完整模板见 `docs/EXECUTION_FLOW_TEMPLATE.md`。

每个阶段有固定的输出格式：

- Phase 1: 命令组表格 + 命令表格
- Phase 2: 步骤表格（步骤/操作/命令/依赖）
- Phase 3: 每步的坐标、依赖元素、拓扑关系
- Phase 4: 依赖校验表格 + 循环依赖检测结果
- Phase 5: 带注释的 bash 命令块
